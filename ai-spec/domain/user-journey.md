# user-journey.md — 用户心理路径设计约束

> 本文件是产品体验层的最高约束。所有前端页面、AI 文案、引导流程必须遵循本文件。
> 核心原则：每一步只推动"半步"，不推销、不制造恐慌、不强制决策。

## 〇、应用导航结构

### TabBar 导航

应用底部固定两个 Tab 页，构成全局导航框架：

| Tab | 页面路径 | 标题 | 说明 |
|-----|---------|------|------|
| 首页 | `pages/home/index` | 优化家 | 漏斗主入口，三态展示 |
| 我的 | `pages/mine/index` | 我的 | 个人中心、报告、设置 |

TabBar 配色：未选中 `#94A3B8`，选中 `#1B6DB2`，白色背景。

### Home 页三态逻辑

首页根据用户登录状态和漏斗进度展示三种状态：

```
状态判断逻辑：
  IF 未登录 OR currentStep <= 1:
    state = "new"         → 新用户态
  ELSE IF currentStep >= 9:
    state = "completed"   → 已完成态
  ELSE:
    state = "inProgress"  → 进行中态

新用户态（new）：
  - 品牌展示（优化家图标 + 品牌名）
  - 主标题："看看你是否正在多付利息"
  - 副标题："1分钟检查，不需要提供个人信息"
  - 三个特性亮点（隐私保护、即时反馈、优化空间）
  - CTA："开始检查" → 跳转 Page 1
  - 底部声明："数据加密传输与存储，仅用于债务优化分析"

进行中态（inProgress）：
  - 标题："继续你的评估"
  - 进度百分比 = (currentStep - 1) / 8 * 100%
  - 进度卡片显示：当前步骤名、优化评分（如有）、压力指数（如有）
  - CTA："继续评估"
    跳转逻辑（数据感知）：
      IF step >= 3 且压力检测未完成 → 回退到 Page 2
      IF step >= 4 且无债务数据 → 回退到 Page 3
      OTHERWISE → 跳转到 currentStep 对应页面
  - 底部提示："数据已自动保存，随时可以继续"

已完成态（completed）：
  - 完成图标 + "评估已完成"
  - 显示优化评分（如有）
  - CTA："查看进度" → 跳转 Page 9
  - 次要按钮："重新评估" → 重置漏斗，跳转 Page 1
```

### Onboarding 引导页

首次打开应用时展示（通过 `onboardingDone` 本地存储标记控制，仅触发一次）。

```
页面路径：pages/onboarding/index
触发时机：Home 页 onShow 检测，若 onboardingDone 为空则跳转
存储标记：uni.setStorageSync('onboardingDone', '1')

三屏引导滑动：
  第 1 屏："看看你是否多付了利息"
    说明：输入月供和收入，1分钟生成月供压力指数，发现潜在优化空间
  第 2 屏："3步完成：检测 → 分析 → 行动"
    说明：从压力检测到债务录入，再到生成个性化优化报告，全程引导
  第 3 屏："数据安全，不接触资金"
    说明：所有数据加密传输和存储，仅用于分析，不涉及任何资金操作

交互：
  - 顶部右上角"跳过"按钮（始终可见）
  - 底部指示器圆点（当前屏高亮）
  - 前两屏底部显示"下一步"文字按钮
  - 第三屏底部显示"开始使用"主按钮
  - 完成/跳过后标记 onboardingDone，switchTab 回首页
```

## 一、设计哲学

```
我们不是卖低利率。
我们卖的是：认知清晰 + 决策安全 + 行动可控。

用户不是因为利率低而行动。
用户是因为："我确定这样做不会更糟。"
```

### 三条转化铁律

| 序号 | 原则 | 心理依据 |
|------|------|----------|
| 1 | 先制造损失感 | 人对损失的敏感度 > 对收益的敏感度（前景理论） |
| 2 | 再给可控希望 | 没有希望只会逃避 |
| 3 | 再给安全路径 | 没有安全就不会行动 |

---

## 二、用户心理阶段模型

高利率用户通常处于 5 个心理阶段，产品流程必须匹配每个阶段：

| 阶段 | 心理状态 | 风险 | 产品任务 |
|------|----------|------|----------|
| 1 | 焦虑但回避 | 不愿录入 | 降低戒备，用"检查"替代"测评" |
| 2 | 想知道情况 | 愿意尝试 | 轻量录入，建立沉没成本 |
| 3 | 看到问题 | 震惊/防御 | 可视化冲击，模糊焦虑→具体损失 |
| 4 | 想改变 | 需要安全感 | 给控制感，把恐慌转为可控 |
| 5 | 决策行动 | 害怕失败 | 分层行动，每步可退出 |

## 三、页面路径与心理映射（9 页渐进式漏斗）

### Page 1 → 安全进入（降低戒备）

```
心理阶段：焦虑但回避
页面目标：让用户愿意开始

✗ 不要写："债务重组测评"
✓ 要写："看看你是否正在多付利息"

✗ 不要写："立即评估你的债务"
✓ 要写："1分钟检查，不需要提供个人信息"

心理逻辑：
  不是让他承认"负债问题"
  而是让他做一个"检查"
  
设计约束：
  - 无需登录即可开始
  - 不出现"债务""负债""重组"等刺激词
  - 主色调温和，不用红色告警色
```

### Page 2 → 快速压力检测（轻量录入）

```
心理阶段：想知道情况
页面目标：建立投入感（沉没成本）

只问两个数据：
  1. 当前每月总还款大概多少？（滑块选择，非精确输入）
  2. 月收入大概多少？（区间选择，非精确数字）

立刻生成：
  "初步压力指数"（月供/收入 的可视化仪表盘）

心理机制：
  一旦输入了第一笔数据，沉没成本开始形成
  用户会想"既然都填了，不如看看结果"

设计约束：
  - 输入方式必须是滑块/选择，不是输入框
  - 立即反馈，不要"提交后等待"
  - 压力指数用中性词（"偏高""较重"），不用"危险""严重"
```

### Page 3 → 完整债务录入

```
心理阶段：从好奇到投入
页面目标：收集完整数据

此时用户已有初步结果，有动机录入更多
引导语："录入具体债务，获得精确分析"

支持三种录入方式（降低阻力）：
  1. 手动表单（兜底）
  2. 拍照识别（OCR，最低阻力）
  3. 快速模板（选择常见产品，预填机构名+类型）

设计约束：
  - 必须支持"先录一笔看效果"，不强制录完所有
  - 每录一笔实时更新分析预览
  - 录入过程中始终显示"已录入 X 笔，已发现 Y 元潜在节省"
```

### Page 4 → 损失可视化报告（制造觉醒）

```
心理阶段：看到问题（震惊/防御）
页面目标：从"模糊焦虑"变成"具体损失"

必须展示的三个核心数据：
  1. 当前加权 APR（与市场均值对比）
  2. 三年多付利息总额（最具冲击力的数字）
  3. 月供占收入比例（与健康线对比）

强视觉表达：
  "如果维持当前结构，3 年将多支付 82,400 元"
  这个数字必须大字体、高对比度、有动画过渡

心理触发点：
  不是告诉他"你有问题"
  而是告诉他"你正在损失多少钱"

设计约束：
  - 金额用累计值（82400 比 每月2289 更震撼）
  - 用与"日常消费"的类比（"相当于 XX 个月房租"）
  - 不要用红色/警告图标制造恐慌
  - 用蓝色/橙色的"信息提示"风格
  
⚠️ 关键：这里绝不能出现任何"立即申请"按钮
    CTA 只能是："看看我的优化空间 →"
```

### Page 5 → 优化空间提示（控制感恢复）

```
心理阶段：想改变但害怕（最易流失的阶段）
页面目标：把"恐慌"转为"可控"

第一句话必须是正面的：
  "好消息是，你有优化空间。"

然后给三个确定性信息：
  1. 成功概率预估（基于评分模型）
  2. 改善幅度模拟（当前 vs 优化后的月供差）
  3. 分阶段路径预览（不是一步到位）

心理机制：
  Page 4 制造了"损失感"（推力）
  Page 5 必须立刻给"可控感"（拉力）
  两者间隔不能超过 1 次滑动

设计约束：
  - 对评分 < 60 的用户，不要说"重组推荐"
    改说："当前更适合优化信用结构"
    并给 30 天行动计划（而非直接推重组）
  - 展示"像你这样的用户，XX% 成功改善了财务状况"
  - 永远不给"0% 成功率"，最低也是"通过调整后有机会改善"
```

### Page 6 → 利率模拟器（强化动机）

```
心理阶段：动机增强
页面目标：让用户"自己看到"收益，而非被告知

核心交互：滑动利率模拟器
  - 左端：当前加权 APR（如 24%）
  - 右端：可能的优化 APR（如 12%）
  - 滑动时实时显示：
      月供变化（24800 → 18200）
      三年节省总额（0 → 82400）
      月供占收入比变化（65% → 48%）

心理强化：
  人脑对"变化"更敏感
  自己拖动滑块看到的效果 > 被文字告知
  这就是"宜家效应" —— 参与创造的东西更有价值

设计约束：
  - 滑块必须有物理阻尼感（不能一滑到底）
  - 数字变化必须有过渡动画
  - 不要预设目标利率，让用户自己探索
  - 底部小字："实际利率取决于个人信用状况和金融机构审核"
```

### Page 7 → 风险评估页（解除最大恐惧）

```
心理阶段：决策前的犹豫
页面目标：风险透明化 → 反而更安心

必须展示的"申请影响评估"：
  1. 是否多查征信 → "优化方案不额外查询征信"
  2. 是否影响信用评分 → "仅在正式申请阶段才会影响"
  3. 失败风险 → "方案评估不产生任何记录"
  4. 费用透明 → "评估阶段完全免费"

心理机制：
  当风险被明示，反而更安心
  未知 >> 已知的坏消息 >> 已知的好消息
  人最恐惧的是"不确定性"，不是"坏结果"

设计约束：
  - 用 Q&A 形式呈现（用户最常问的问题）
  - 每个答案必须具体、不含糊
  - 不说"一般不会"，要说"在 XX 情况下不会"
  - 这个页面不要求任何输入，纯信息展示
```

### Page 8 → 分层行动页（降低门槛）

```
心理阶段：准备行动但害怕失败
页面目标：把"大决策"拆成"小步骤"

不要一个"立即申请"按钮。改为四层递进：

  第一层：生成资料清单
    "看看申请需要准备什么"
    → 输出：所需文件列表 + 预估准备时间
    → 用户可退出

  第二层：生成申请材料
    "一键整理你的申请资料"
    → 输出：结构化的申请文档
    → 用户可退出

  第三层：模拟审批结果（PreAudit 预审）
    "预审一下通过概率"
    → 调用 PreAuditEngine 估算通过概率（35%-92%）
    → 输出：概率百分比 + 最多 3 条个性化建议
    → 前端同步实现 estimatePreAudit() 函数（preaudit.js），
      与后端 PreAuditEngine 使用相同规则配置
    → 用户可退出

  第四层：正式提交（V2.0 功能）
    "确认提交申请"
    → 此步才是不可逆操作

心理机制：
  每一层都是"可退出"的
  每一层完成后都有"成就感"
  到第四层时，用户已经投入了足够的沉没成本

设计约束：
  - 每层之间必须有明确的"暂不继续"出口
  - 每层完成后展示进度条（1/4 → 2/4 → 3/4）
  - 不自动跳到下一层，必须用户主动点击
  - 已完成的层可以随时回看
```

### Page 9 → 持续陪伴页

```
心理阶段：行动后的不安
页面目标：降低"后悔感"，建立长期关系

展示内容：
  - 你的优化进度追踪
  - 30/60/90 天行动计划 checklist
  - 下一个检查点提醒
  - "你已经迈出了第一步"正面强化
  - ConsultCard 咨询预约卡片（见下方说明）

ConsultCard — 咨询预约组件：
  位置：Page 9 底部，标题"需要专人协助？"
  功能：用户可留下联系方式预约 1 对 1 咨询
  交互流程：
    1. 选择咨询类型（债务优化 / 利率协商 / 综合咨询）
    2. 填写/确认联系电话（已登录用户预填脱敏手机号，可修改）
    3. 补充说明（选填，最多 200 字）
    4. 点击"预约咨询" → 调用 consultation API 创建咨询请求
    5. 提交成功后切换为成功状态：显示咨询类型、联系电话确认信息
       提示："我们将在 1 个工作日内通过电话联系你"
  设计约束：
    - 手机号验证：/^1[3-9]\d{9}$/
    - 已登录用户默认使用注册手机号，显示脱敏格式，可切换为手动输入
    - 提交后不可重复提交（按钮区域替换为成功状态展示）

设计约束：
  - 这不是一个"结果页"，而是"陪伴页"
  - 定期推送微小进展（"本月月供已降低 XX 元"）
  - 评分 < 60 的用户在这里获得"信用修复路线图"
```

## 五、评分 < 60 用户的特殊路径

```
⚠️ 绝对不能让这类用户"申请失败"

替代方案：
  Page 5 展示："当前更适合优化信用结构"
  Page 6 展示："信用修复路线图"（替代利率模拟器）
  Page 8 变为：
    第一层：生成 30 天改善计划
    第二层：设置还款提醒
    第三层：30 天后重新评估

心理逻辑：
  一次"申请失败"的体验 → 永远流失
  一次"暂时不适合，但有路径"的体验 → 30天后回来

产品指标：
  评分 < 60 用户的 30 天回访率 > 40%（目标）
```

## 六、文案语气约束（全局适用）

### 绝对禁止的表达

| 禁止 | 替代 |
|------|------|
| "你的债务问题很严重" | "你的财务结构有优化空间" |
| "你需要立即处理" | "越早调整，节省越多" |
| "高风险" | "需要关注" |
| "负债过高" | "月供压力偏大" |
| "申请失败" | "暂时不适合，建议先优化" |
| "你的征信可能有问题" | "信用状况有提升空间" |
| "赶紧行动" | "你可以从这一步开始" |
| "最后机会" | 永远不使用 |

### 必须遵循的语气原则

```
1. 主语是"你的财务结构"，不是"你"
   ✓ "你的月供结构偏重" ✗ "你月供太多了"

2. 动词用"优化/调整/改善"，不用"解决/修复/治疗"
   财务问题不是"病"

3. 数字用"节省 XX 元"，不用"多付 XX 元"做反复强调
   Page 4 制造一次冲击即可，后续页面切换到正面框架

4. 时间框架用"90天"，不用"立刻/马上"
   给用户心理缓冲空间

5. 对比用"当前 vs 优化后"，不用"你 vs 别人"
   不制造社交比较压力
```

## 七、AI 文案生成约束（注入 suggestion-gen.md）

```
当 AI 生成优化建议时，必须遵循以下心理路径：

第一段：确认感受（共情）
  "管理多笔债务确实需要花费精力..."

第二段：量化损失（仅一次）
  "按当前结构，未来3年将多支付约 XX 元利息"

第三段：立即转正面
  "好消息是，通过调整优先级，你可以..."

第四段：具体行动（小步骤）
  "第一步：优先处理 {creditor}（节省效果最显著）"

第五段：安全兜底
  "这些调整不影响你的信用记录，随时可以调整节奏"

禁止：
  - 连续两段都在讲"损失/风险/问题"
  - 用"必须""一定""否则"等强制性词汇
  - 与其他用户做比较
```

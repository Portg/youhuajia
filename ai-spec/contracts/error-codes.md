# error-codes.md — 统一错误码定义

> 所有接口返回的错误码必须从本文件中选取，不得自行编造。

---

## 编码规则

```
格式：XYYZZZ
  X   = 类别（4=业务异常, 5=系统异常）
  YY  = 模块（01=认证, 02=债务, 03=画像, 04=引擎, 05=AI, 06=报告）
  ZZZ = 序号
```

### HTTP 状态码映射（遵循 Google AIP-193）

| 业务错误类型 | HTTP Status | Google Status | 说明 |
|-------------|-------------|---------------|------|
| 参数校验失败 | 400 | INVALID_ARGUMENT | 入参格式、范围错误 |
| 未认证 | 401 | UNAUTHENTICATED | Token 缺失、过期、无效 |
| 无权限 | 403 | PERMISSION_DENIED | 访问其他用户数据 |
| 资源不存在 | 404 | NOT_FOUND | 债务/报告/任务不存在 |
| 状态冲突 | 409 | ALREADY_EXISTS / ABORTED | 重复创建、乐观锁冲突 |
| 前置条件不满足 | 400 | FAILED_PRECONDITION | 状态机校验不通过 |
| 资源超限 | 429 | RESOURCE_EXHAUSTED | 频率限制、数量上限 |
| 服务端异常 | 500 | INTERNAL | 系统级错误 |
| 外部服务不可用 | 503 | UNAVAILABLE | AI/存储服务宕机 |

## Java 使用方式

```java
// 错误码枚举（含 Google 风格 Status 映射）
public enum ErrorCode {

    SUCCESS(200, "success", 200, "OK"),

    // 认证
    SMS_RATE_LIMIT(401001, "验证码发送频率过高，请稍后重试", 429, "RESOURCE_EXHAUSTED"),
    SMS_CODE_INVALID(401002, "验证码错误或已过期", 400, "INVALID_ARGUMENT"),
    // ...

    // 债务
    DEBT_NOT_FOUND(402001, "债务记录不存在", 404, "NOT_FOUND"),
    DEBT_PRINCIPAL_INVALID(402002, "本金必须大于零", 400, "INVALID_ARGUMENT"),
    DEBT_STATE_INVALID(402006, "当前状态不允许此操作", 400, "FAILED_PRECONDITION"),
    DEBT_VERSION_CONFLICT(402007, "数据已被其他操作修改，请刷新后重试", 409, "ABORTED"),
    // ...

    private final int code;        // 业务错误码
    private final String message;  // 错误信息
    private final int httpStatus;  // HTTP 状态码
    private final String status;   // Google 风格状态枚举
}

// 使用
throw new BizException(ErrorCode.DEBT_NOT_FOUND);
throw new BizException(ErrorCode.DEBT_VERSION_CONFLICT);

// 统一错误响应体（对齐 google.rpc.Status）
{
  "error": {
    "code": 402001,
    "message": "债务记录不存在",
    "status": "NOT_FOUND",
    "details": []
  },
  "traceId": "a1b2c3d4e5f6"
}
```

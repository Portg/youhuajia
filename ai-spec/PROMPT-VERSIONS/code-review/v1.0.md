# code-review v1.0

# 生效日期：2026-03-09 | 初始版本

## 意图

以 intent.md Anti-Goals 为基准对 AI 生成的代码执行 Review。
不是语法检查——重点是业务规则合规性和金融场景安全性。

## 上下文加载顺序（CONTEXT-SLICE 切片 D）

1. `@ai-spec/domain/intent.md`（Anti-Goals 逐条核对）
2. `@ai-spec/VIBE-CHECKLIST.md`（六关门禁）
3. `@CLAUDE.md`（禁止项 + 命名规范）
4. `@ai-spec/SCRATCHPAD.md`（最近 3 条记录）
5. [被审查的代码文件]
6. [对应的测试文件]

## 任务描述

执行六关门禁检查 + ADR 合规性 + 测试完整性 + Prompt 可追溯性

## 检查项

### ADR 合规性
- [ ] 依赖是否违反技术选型约束（MyBatis-Plus, not JPA）？
- [ ] 代码结构是否符合分层架构（Controller→Service→Mapper）？
- [ ] 有没有 AI 绕过了某个重要决策？

### 测试完整性
- [ ] Anti-Goals 中每条规则是否有对应测试方法？
- [ ] 测试方法名是否包含业务语义（@DisplayName）？
- [ ] 没有空测试或 @Disabled？

### @AiGenerated 标注
- [ ] 复杂逻辑是否已标注？
- [ ] reviewBy 日期是否合理？

## 输出格式

按三级分类输出问题报告：
1. **严重**：违反 Anti-Goals / BigDecimal 缺失 / 敏感数据泄露
2. **一般**：命名不规范 / 过期 @AiGenerated / 缺少测试
3. **建议**：代码风格 / 重构信号 / 性能优化

# generate-api v1.0

# 生效日期：2026-03-09 | 初始版本

## 意图

生成符合优化家金融系统安全规范的 REST 接口层。
不是简单的 CRUD 脚手架——需要理解业务规则后再生成。

## 上下文加载顺序

1. `@ai-spec/domain/intent.md`（Anti-Goals + 数据敏感级别）
2. `@ai-spec/contracts/openapi.yaml`（仅本次涉及路径）
3. `@ai-spec/domain/state-machines.yaml`
4. `@CLAUDE.md`
5. `@ai-spec/TEST-SPEC.md`（测试要求）

## 任务描述

生成：Controller + Service 接口 + DTO + 对应测试骨架

## 验收条件

- [ ] Controller 中不含任何 if/else 业务判断
- [ ] 所有入参有 @Valid + 对应约束注解
- [ ] 返回结构严格匹配统一 Response 格式（google.rpc.Status）
- [ ] 金额字段全部使用 BigDecimal
- [ ] 异常处理使用 @ControllerAdvice + BizException(ErrorCode)
- [ ] intent.md Anti-Goals 是否有对应测试用例？
- [ ] 复杂逻辑是否标注了 @AiGenerated？
- [ ] 敏感字段日志已脱敏

## 输出格式

1. 《生成决策说明》（含推理链）
2. 代码文件（Controller / Service / DTO）
3. 测试骨架文件

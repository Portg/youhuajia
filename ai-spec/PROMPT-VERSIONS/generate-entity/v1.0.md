# generate-entity v1.0

# 生效日期：2026-03-09 | 初始版本

## 意图

生成符合优化家数据层规范的 Entity + Mapper + Flyway 迁移脚本。
必须严格遵循 CLAUDE.md 3.1 节数据层约束。

## 上下文加载顺序

1. `@CLAUDE.md`（数据层约束 3.1 节）
2. `@ai-spec/domain/entities.md`
3. `@ai-spec/domain/evolution.md`
4. `@ai-spec/domain/state-machines.yaml`

## 任务描述

生成：Entity 类 + Mapper 接口 + Flyway 迁移脚本 + 对应测试

## 验收条件

- [ ] 金额字段使用 BigDecimal + DECIMAL(18,4)
- [ ] 比率字段使用 BigDecimal + DECIMAL(10,6)
- [ ] 所有表含 id / create_time / update_time / deleted
- [ ] 并发控制表含 version 字段 + @Version 注解
- [ ] 枚举持久化使用 @EnumValue 存字符串值
- [ ] 索引命名 idx_{表名}_{字段名}，唯一约束 uk_{表名}_{字段名}
- [ ] 状态字段与 state-machines.yaml 对齐
- [ ] 迁移脚本文件名符合 Flyway 命名规范（V{版本}__{描述}.sql）

## 输出格式

1. 《生成决策说明》
2. Entity 类
3. Mapper 接口
4. Flyway 迁移脚本
5. Mapper 测试（核心查询）

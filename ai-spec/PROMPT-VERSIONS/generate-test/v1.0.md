# generate-test v1.0

# 生效日期：2026-03-09 | 初始版本

## 意图

为已生成的业务代码补全测试套件，
确保 Anti-Goals 中每条规则都有对应的失败场景测试。

## 上下文加载顺序（CONTEXT-SLICE 切片 G）

1. `@ai-spec/domain/intent.md`（Anti-Goals 全文）
2. `@ai-spec/domain/state-machines.yaml`
3. `@ai-spec/engine/rules.md`
4. `@ai-spec/TEST-SPEC.md`
5. [待测试的代码文件]

## 任务描述

生成：JUnit5 测试类，覆盖正常路径 + Anti-Goals + 状态机边界

## 验收条件

- [ ] 每条 Anti-Goal 都有对应的 @Test 方法，方法名包含业务语义
- [ ] 所有非法状态跳跃都有 assertThrows 测试
- [ ] Service 层覆盖率达到 90% 以上
- [ ] 没有 @Disabled 或空测试方法
- [ ] 测试使用 @DisplayName 包含中文业务描述
- [ ] 方法命名：should_{结果}_when_{条件}

## 输出格式

1. 测试类文件
2. 覆盖率报告（列出已覆盖的 Anti-Goals 编号）

# /scratchpad — 跨会话连续性记录

在会话结束前生成 SCRATCHPAD.md 条目，确保关键决策和推理链不丢失。

## 触发方式

```
/scratchpad
```

---

## 执行流程

### 第一步：回顾本次会话

自动汇总本次会话中的：
- 完成的任务列表
- 做出的关键决策（如技术选型、方案选择、架构调整）
- 放弃的方案及原因
- 使用的 Prompt 版本（如适用）
- 遇到的问题和解决方式

### 第二步：生成推理链

对每个关键决策，按标准格式输出推理过程：

```
决策：[决策名称]
1. 我读取了 [文件/规则]
2. 我注意到 [关键约束，引用原文]
3. 我考虑了 [方案A] 和 [方案B]
4. 方案A 的问题：[具体原因]
5. 因此我选择了 [方案B]，因为 [与约束的对应关系]
```

### 第三步：识别遗留问题

从会话中提取：
- 未完成的任务（标记 `- [ ]`）
- 需要确认的设计决策
- 需要人工复核的 @AiGenerated 代码
- 建议的 spec 变更

### 第四步：写入 SCRATCHPAD.md

读取 `ai-spec/SCRATCHPAD.md`，将新记录插入到 `<!-- 新记录插入此处 -->` 之后。

记录格式：
```markdown
---
日期：{今天日期} 任务类型：{任务类型} 关联切片：{使用的切片}
使用的 Prompt 版本：{如适用}

### 本次完成了什么
- {完成项1}
- {完成项2}

### 关键决策
| 决策内容 | 选择方案 | 原因摘要 |
|----------|----------|----------|

### AI 推理链（关键决策必填）
{推理链内容}

### 放弃的方案
| 方案描述 | 放弃原因 |
|----------|----------|

### 遗留问题（下次会话继续）
- [ ] {遗留问题}

### Spec 变更建议
- 文件：{文件名} 内容：{变更描述}

### Prompt 效果反馈
版本：{版本号} 评价：{好用/有问题} 具体：{描述}
---
```

### 第五步：条目压缩

如果 SCRATCHPAD.md 中已有超过 10 条记录，自动将最老的 7 条压缩为「历史摘要」段落。

### 第六步：确认

向用户展示生成的记录内容，确认后写入文件。
